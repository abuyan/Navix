# Navix 产品升级规划书 (Roadmap)

## 1. 产品愿景 (Vision)
**Navix** 不仅仅是一个导航站，而是一个**“高审美的、AI 赋能的、支持高度定制化的个人知识门户”**。

它通过 **“私域工具（个人桌面/导入）”** 留住用户，通过 **“公域内容（周刊/资讯）”** 吸引流量。

---

## 2. 核心竞争力 (Unique Selling Point)
*   **高度自定义：** 支持浏览器书签导入，解决跨设备同步痛点。
*   **严选内容：** 拒绝低质量聚合，主打有观点的“周刊”推荐。
*   **AI 深度集成：** 自动化录入、智能摘要、语义搜索。
*   **现代交互：** 基于 Next.js + Tailwind 的卡片式桌面体验，支持自由拖拽。

---

## 3. 功能模块规划

### 阶段一：基础功能完善与自定义能力 (MVP+)
*   [ ] **用户系统：** 接入登录功能（NextAuth）。
*   [ ] **导入功能强化：** 优化 `bookmark-parser`，支持大规模 HTML 书签导入并自动分类。
*   [ ] **拖拽排序：** 完善目前的 `DndProvider` 实现，支持持久化保存排序结果。

### 阶段二：AI 赋能录入与摘要 (核心差异化)
*   [ ] **AI 实验室：** 搭建 `/api/ai` 接口，对接 DeepSeek/OpenAI 模型。
*   [ ] **智能录入：** 输入 URL 后，AI 自动抓取并填充 `Title`, `Description` (一句话简介), `Tags` (标签)。
*   [ ] **智能卡片：** 前台展示 AI 提取的关键词标签。

### 阶段三：内容系统 (Notion + AI 自动化中台)
*   **架构核心：** 使用 Notion 作为 Headless CMS（内容管理后台），实现“一次生产，多处分发”。
*   **数据流设计：**
    1.  **素材收集 (Notion DB A)：** 用户日常通过 Clipper 剪藏素材。
    2.  **AI 编排 (脚本)：** 定时读取素材库 -> AI 自动生成周刊草稿 (含官网版、小红书文案版) -> 写入 Notion 周刊库 (DB B)。
    3.  **人工审核：** 在 Notion 中校对并标记 "Published"。
    4.  **多端分发：** 
        *   **官网：** Next.js 通过 API 拉取渲染。
        *   **社媒：** 手动复制优化后的文案至公众号/小红书。
*   **技术栈：** Notion API, `react-notion-x` (渲染), DeepSeek API (生成)。

---

## 5. 自动化工作流说明 (Weekly Newsletter Workflow)

目前已实现“一键生成、多端分发”的自动化写作流程：

### 5.1 准备工作
1.  **Notion 素材库**：在素材库中勾选 **“周刊”** 复选框（建议每周选 3-5 条）。
2.  **环境变量**：确保 `.env.local` 中配置了 `DEEPSEEK_API_KEY`、`NOTION_TOKEN` 以及两个数据库 ID。

### 5.2 运行指令
在终端输入以下命令即可启动 AI 助手：
```bash
npm run weekly
```

### 5.3 自动化逻辑
1.  **扫描**：脚本自动抓取 Notion 中勾选了“周刊”的素材。
2.  **生成**：DeepSeek AI 扮演“产品经理阿布”的人设，生成 **网站/博客版**（深度点评）和 **小红书版**（情绪化种草）。
3.  **分发**：
    *   **本地备份**：生成 `.md` 文件保存在 `docs/` 目录下。
    *   **Notion 回写**：自动在“周刊数据库”创建新页面，填入日期、自动勾选平台，并排版好正文。
